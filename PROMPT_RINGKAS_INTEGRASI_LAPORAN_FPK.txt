PROMPT INTEGRASI LAPORAN FPK KE WEBSITE EXISTING
================================================

Bertindaklah sebagai Senior Laravel Full-Stack Engineer dan Content Migration Engineer.

KONTEKS
-------

Project website FPK Kota Malang sudah hampir selesai. Website publik, autentikasi admin,
panel admin, database, route, tampilan, dan sebagian besar fitur sudah tersedia.

File sumber berada di:

`references/Laporan_Profil_FPK_Kota_Malang_2026.docx`

Tugas ini adalah INTEGRASI DATA DAN GAMBAR ke project yang SUDAH ADA, bukan rebuild.

ATURAN UTAMA
------------

1. Baca `README.md`.
2. Audit source code aktual untuk memahami progres project.
3. Jangan bergantung pada file `.md` lain.
4. Jangan menghapus, mengganti, atau merombak fitur existing.
5. Jangan mengubah route, URL, desain utama, dark mode, autentikasi, panel admin,
   struktur one-page, CRUD, atau database tanpa kebutuhan jelas.
6. Jangan menjalankan:
   - `php artisan migrate:fresh`
   - `php artisan db:wipe`
   - reset database
7. Jangan mengedit migration lama yang sudah digunakan. Jika perlu schema baru,
   buat migration additive baru.
8. Jangan menimpa data existing secara sembarangan.
9. Jangan mengarang kontak, media sosial, artikel, agenda, tanggal, lokasi, nama,
   gelar, atau informasi lain yang tidak tersedia.
10. Gunakan patch minimum dan pertahankan backward compatibility.

LANGKAH KERJA
-------------

### 1. Audit project

Periksa:

- routes
- controllers
- requests
- models
- migrations
- seeders
- Blade views
- admin panel
- autentikasi
- storage/media
- tests

Pahami sendiri:

### 2. Baca file DOCX

Baca seluruh isi:

`references/Laporan_Profil_FPK_Kota_Malang_2026.docx`

Ekstrak bila perlu:

- teks
- tabel
- gambar dari `word/media/`

Jangan memindahkan file DOCX ke public storage.

### 3. Petakan ke fitur existing

Gunakan data dokumen untuk fitur yang sudah tersedia:

**Profil FPK**
- pengertian
- latar belakang
- konteks kebangsaan
- tujuan
- peran
- tugas pokok
- dasar hukum
- masa bakti

**Beranda/Hero**
- nama organisasi
- tagline
- ringkasan
- dasar hukum
- masa bakti
- gambar hero bila layak

**Pengurus**
- unsur pimpinan
- empat bidang kerja
- jabatan
- nama
- gelar
- urutan
- periode 2025–2027
- biro pendukung hanya jika schema existing dapat menampungnya

**Artikel**
- jangan otomatis membuat artikel published dari dokumentasi kegiatan
- boleh membuat draft hanya jika datanya cukup

**Agenda**
- jangan mengubah kegiatan lama menjadi agenda mendatang

**Kontak**
- jangan mengubah jika dokumen tidak menyediakan data resmi

### 4. Seleksi gambar

Ekstrak gambar ke folder kerja non-public.

Klasifikasikan:

- logo
- foto pengurus
- foto kegiatan
- infografik
- screenshot slide
- tidak layak digunakan

Gunakan gambar hanya jika:

- relevan
- cukup tajam
- tidak blur
- tidak penuh teks slide
- tidak merusak desain
- sesuai rasio
- aman dipublikasikan

Jangan gunakan sebagai hero:

- screenshot halaman laporan
- screenshot slide penuh
- infografik penuh teks
- gambar blur
- foto berkualitas rendah

Prioritas hero:

1. foto kegiatan landscape terbaik
2. foto kelompok resmi
3. logo resmi
4. fallback existing

Jangan mengganti gambar existing jika kualitas existing lebih baik.

### 5. Perlindungan data existing

Gunakan aturan:

- create jika belum ada
- update hanya field kosong atau jelas lebih tepat
- skip jika konflik
- pertahankan data existing jika lebih lengkap
- jangan mengubah slug
- jangan mengubah status published
- jangan menghapus data yang tidak ada di DOCX
- jangan mengganti kontak existing
- jangan mengganti media existing tanpa alasan

### 6. Implementasi

Lakukan integrasi minimum ke sistem yang sudah ada.

Jangan membuat modul baru jika fitur existing sudah dapat menampung data.

Pertahankan:

- desain
- navbar
- one-page homepage
- light/dark mode
- admin sidebar
- route
- alur CRUD
- upload system existing

Boleh:

- mengisi data profil
- mengisi struktur pengurus
- mengganti placeholder dengan gambar yang lebih layak
- memperbaiki crop, alt text, dan fallback
- menambah migration additive jika benar-benar diperlukan

### 7. Validasi

Pastikan:

- website publik tetap berjalan
- panel admin tetap berjalan
- tidak ada route rusak
- data existing tidak hilang
- gambar tidak broken
- dark mode tetap bekerja
- responsive tetap baik
- tidak ada N+1 baru
- data belum terverifikasi tidak otomatis dipublikasikan

Jalankan jika tersedia:

```bash
php artisan route:list
php artisan test
./vendor/bin/pint --test
npm run build
```
